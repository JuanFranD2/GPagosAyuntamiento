getFont ( $familyRaw,
$subtypeRaw = "normal" )

Resolves a font family & subtype into an actual font file Subtype can be one of 'normal', 'bold', 'italic' or 'bold_italic'. If the particular font family has no suitable font file, the default font (Options::defaultFont) is used. The font file returned is the absolute pathname to the font file on the system.

◆ mapTextToFonts()

mapTextToFonts ( string $text,
array $fontFamilies,
string $subtype = "normal",
int $count = -1,
bool $returnSubstring = false )

Maps substrings of text against the provided font list. This is achieved by parsing each character of the string against the supported glyphs for each font. Fonts preference is based on the order of the font list.

Returns an array containing substring information that indicates the matched font (if any), start index, substring length, and (optionally) the actual text of the substring.

Parameters
string$textThe text to map
array$fontFamiliesList of font families to map against
string$subtypeThe font subtype (italic, bold, etc.)
int$countThe number of matches to return
bool$returnSubstringShould the actual matched text be returned
Returns

◆ USER_FONTS_FILE

const USER_FONTS_FILE = "installed-fonts.json"

Name of the user font families file

This file must be writable by the webserver process only to update it with save_font_families() after adding the .afm file references of a new font family with FontMetrics::saveFontFamilies(). This is typically done only from command line with load_font.php on converting ttf fonts to ufm with php-font-lib.
