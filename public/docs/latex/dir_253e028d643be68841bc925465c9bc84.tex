\hypertarget{readme.md_autotoc_md477}{}\doxysubsubsection{\texorpdfstring{Overview}{Overview}}\label{readme.md_autotoc_md477}
Type\+Box is a runtime type builder that creates in-\/memory Json \doxylink{class_schema}{Schema} objects that infer as Type\+Script types. The schematics produced by this library are designed to match the static type checking rules of the Type\+Script compiler. Type\+Box offers a unified type that can be statically checked by Type\+Script and runtime asserted using standard Json \doxylink{class_schema}{Schema} validation.

This library is designed to allow Json \doxylink{class_schema}{Schema} to compose similar to how types compose within Type\+Script\textquotesingle{}s type system. It can be used as a simple tool to build up complex schematics or integrated into REST and RPC services to help validate data received over the wire.

\hypertarget{readme.md_autotoc_md480}{}\doxysubsubsection{\texorpdfstring{Types}{Types}}\label{readme.md_autotoc_md480}
Type\+Box types are Json \doxylink{class_schema}{Schema} fragments that compose into more complex types. Each fragment is structured such that any Json \doxylink{class_schema}{Schema} compliant validator can runtime assert a value the same way Type\+Script will statically assert a type. Type\+Box offers a set of Json Types which are used to create Json \doxylink{class_schema}{Schema} compliant schematics as well as a Java\+Script type set used to create schematics for constructs native to Java\+Script.

These types can not be used with any standard Json \doxylink{class_schema}{Schema} validator; but can be used to frame schematics for interfaces that may receive Json validated data. Java\+Script types are prefixed with the {\ttfamily \mbox{[}Java\+Script\mbox{]}} JSDoc comment for convenience. The following table lists the supported types.

\hypertarget{readme.md_autotoc_md525}{}\doxysubsubsubsection{\texorpdfstring{Type\+Compiler}{Type\+Compiler}}\label{readme.md_autotoc_md525}
The Type\+Box Type\+Compiler is a high performance JIT validation compiler that transforms Type\+Box types into optimized Java\+Script validation routines. The compiler is tuned for fast compilation as well as fast value assertion. It is built to serve as a validation backend that can be integrated into larger applications. It can also be used for code generation.

The Type\+Compiler is provided as an optional import.

Use the Compile function to JIT compile a type. Note that compilation is generally an expensive operation and should only be performed once per type during application start up. Type\+Box does not cache previously compiled types, and applications are expected to hold references to each compiled type for the lifetime of the application.

\hypertarget{readme.md_autotoc_md528}{}\doxysubsubsection{\texorpdfstring{Type\+System}{Type\+System}}\label{readme.md_autotoc_md528}
The Type\+Box Type\+System module provides configurations to use either Json \doxylink{class_schema}{Schema} or Type\+Script type checking semantics. Configurations made to the Type\+System module are observed by the Type\+Compiler, Value and Error modules.

\label{readme.md_typesystem-policies}%
\Hypertarget{readme.md_typesystem-policies}%
\hypertarget{readme.md_autotoc_md529}{}\doxysubsubsubsection{\texorpdfstring{Policies}{Policies}}\label{readme.md_autotoc_md529}
Type\+Box validates using standard Json \doxylink{class_schema}{Schema} assertion policies by default. The Type\+System\+Policy module can override some of these to have Type\+Box assert values inline with Type\+Script static checks. It also provides overrides for certain checking rules related to non-\/serializable values (such as void) which can be helpful in Json based protocols such as Json Rpc 2.\+0.
