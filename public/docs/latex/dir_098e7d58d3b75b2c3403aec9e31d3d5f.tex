\hypertarget{README.md_autotoc_md10533}{}\doxysubsubsection{\texorpdfstring{Manipulation}{Manipulation}}\label{README.md_autotoc_md10533}
The resulting data structure consists mainly of five basic types\+: {\ttfamily CSSList}, {\ttfamily Rule\+Set}, {\ttfamily Rule}, {\ttfamily Selector} and {\ttfamily Value}. There are two additional types used\+: {\ttfamily Import} and {\ttfamily Charset}, which you won’t use often.\hypertarget{README.md_autotoc_md10534}{}\doxysubsubsubsection{\texorpdfstring{CSSList}{CSSList}}\label{README.md_autotoc_md10534}
{\ttfamily CSSList} represents a generic CSS container, most likely containing declaration blocks (rule sets with a selector), but it may also contain at-\/rules, charset declarations, etc.

To access the items stored in a {\ttfamily CSSList} – like the document you got back when calling {\ttfamily \$parser-\/\texorpdfstring{$>$}{>}parse()} –, use {\ttfamily get\+Contents()}, then iterate over that collection and use {\ttfamily instanceof} to check whether you’re dealing with another {\ttfamily CSSList}, a {\ttfamily Rule\+Set}, a {\ttfamily Import} or a {\ttfamily Charset}.

To append a new item (selector, media query, etc.) to an existing {\ttfamily CSSList}, construct it using the constructor for this class and use the {\ttfamily append(\$o\+Item)} method.\hypertarget{README.md_autotoc_md10535}{}\doxysubsubsubsection{\texorpdfstring{Rule\+Set}{Rule\+Set}}\label{README.md_autotoc_md10535}
{\ttfamily Rule\+Set} is a container for individual rules. The most common form of a rule set is one constrained by a selector. The following concrete subtypes exist\+:


\begin{DoxyItemize}
\item {\ttfamily At\+Rule\+Set} – for generic at-\/rules for generic at-\/rules which are not covered by specific classes, i.\+e., not {\ttfamily @import}, {\ttfamily @charset} or {\ttfamily @media}. A common example for this is {\ttfamily @font-\/face}.
\item {\ttfamily Declaration\+Block} – a {\ttfamily Rule\+Set} constrained by a {\ttfamily Selector}; contains an array of selector objects (comma-\/separated in the CSS) as well as the rules to be applied to the matching elements.
\end{DoxyItemize}

Note\+: A {\ttfamily CSSList} can contain other {\ttfamily CSSList}s (and {\ttfamily Import}s as well as a {\ttfamily Charset}), while a {\ttfamily Rule\+Set} can only contain {\ttfamily Rule}s.

If you want to manipulate a {\ttfamily Rule\+Set}, use the methods {\ttfamily add\+Rule(Rule \$rule)}, {\ttfamily get\+Rules()} and {\ttfamily remove\+Rule(\$rule)} (which accepts either a {\ttfamily Rule} or a rule name; optionally suffixed by a dash to remove all related rules).\hypertarget{README.md_autotoc_md10536}{}\doxysubsubsubsection{\texorpdfstring{Rule}{Rule}}\label{README.md_autotoc_md10536}
{\ttfamily Rule}s just have a string key (the rule) and a {\ttfamily Value}.\hypertarget{README.md_autotoc_md10537}{}\doxysubsubsubsection{\texorpdfstring{Value}{Value}}\label{README.md_autotoc_md10537}
{\ttfamily Value} is an abstract class that only defines the {\ttfamily render} method. The concrete subclasses for atomic value types are\+:


\begin{DoxyItemize}
\item {\ttfamily Size} – consists of a numeric {\ttfamily size} value and a unit.
\item {\ttfamily Color} – colors can be input in the form \#rrggbb, \#rgb or schema(val1, val2, …) but are always stored as an array of (\textquotesingle{}s\textquotesingle{} =\texorpdfstring{$>$}{>} val1, \textquotesingle{}c\textquotesingle{} =\texorpdfstring{$>$}{>} val2, \textquotesingle{}h\textquotesingle{} =\texorpdfstring{$>$}{>} val3, …) and output in the second form.
\item {\ttfamily CSSString} – this is just a wrapper for quoted strings to distinguish them from keywords; always output with double quotes.
\item {\ttfamily \doxylink{class_u_r_l}{URL}} – URLs in CSS; always output in {\ttfamily \doxylink{class_u_r_l}{URL}("{}"{})} notation.
\end{DoxyItemize}

There is another abstract subclass of {\ttfamily Value}, {\ttfamily Value\+List}\+: A {\ttfamily Value\+List} represents a lists of {\ttfamily Value}s, separated by some separation character (mostly {\ttfamily ,}, whitespace, or {\ttfamily /}).

There are two types of {\ttfamily Value\+List}s\+:


\begin{DoxyItemize}
\item {\ttfamily Rule\+Value\+List} – The default type, used to represent all multivalued rules like {\ttfamily font\+: bold 12px/3 Helvetica, Verdana, sans-\/serif;} (where the value would be a whitespace-\/separated list of the primitive value {\ttfamily bold}, a slash-\/separated list and a comma-\/separated list).
\item {\ttfamily CSSFunction} – A special kind of value that also contains a function name and where the values are the function’s arguments. Also handles equals-\/sign-\/separated argument lists like {\ttfamily filter\+: alpha(opacity=90);}.
\end{DoxyItemize}\hypertarget{README.md_autotoc_md10538}{}\doxysubsubsubsection{\texorpdfstring{Convenience methods}{Convenience methods}}\label{README.md_autotoc_md10538}

\item {\ttfamily get\+All\+Rule\+Sets()} – does what it says; no matter how deeply nested the rule sets are.
