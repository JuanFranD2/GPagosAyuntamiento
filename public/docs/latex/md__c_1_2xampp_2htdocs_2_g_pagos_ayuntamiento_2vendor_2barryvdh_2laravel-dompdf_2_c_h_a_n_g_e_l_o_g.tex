This release updates the config for \href{https://github.com/dompdf/dompdf/releases/tag/v3.1.0}{\texttt{dompdf/dompdf v3.\+1.\+0}} which contains the. following breaking \doxylink{class_u_r_l}{URL}\+:

\begin{quote}
{\bfseries{Breaking Change}} This release adds the "{}data\+://"{} scheme to the protocol validation rules. Installations that explicitly define the allowed protocols but do not include the "{}data\+://"{} protocol will no longer render data-\/\+URIs. This is a change from previous versions, where data-\/\+URIs were not processed through the validated rules. Installations that use the default validation rules included with \doxylink{namespace_dompdf}{Dompdf} should see no impact. \end{quote}
