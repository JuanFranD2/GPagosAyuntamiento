HTML5 is a standards-\/compliant HTML5 parser and writer written entirely in PHP. It is stable and used in many production websites, and has well over \href{https://packagist.org/packages/masterminds/html5}{\texttt{five million downloads}}.

HTML5 provides the following features.


\begin{DoxyItemize}
\item An HTML5 serializer
\item Support for PHP namespaces
\item \doxylink{namespace_composer}{Composer} support
\item Event-\/based (SAX-\/like) parser
\item A DOM tree builder
\item Interoperability with \href{https://github.com/technosophos/querypath}{\texttt{Query\+Path}}
\item Runs on {\bfseries{PHP}} 5.\+3.\+0 or newer
\end{DoxyItemize}

The unit tests exercise each piece of the API, and every public function is well-\/documented.\hypertarget{README.md_autotoc_md9808}{}\doxysubsubsubsection{\texorpdfstring{Parser Design}{Parser Design}}\label{README.md_autotoc_md9808}
The parser is designed as follows\+:


\begin{DoxyItemize}
\item The {\ttfamily Scanner} handles scanning on behalf of the parser.
\item The {\ttfamily Tokenizer} requests data off of the scanner, parses it, clasifies it, and sends it to an {\ttfamily Event\+Handler}. It is a {\itshape recursive descent parser.}
\item The {\ttfamily Event\+Handler} receives notifications and data for each specific semantic event that occurs during tokenization.
\item The {\ttfamily DOMBuilder} is an {\ttfamily Event\+Handler} that listens for tokenizing events and builds a document tree ({\ttfamily DOMDocument}) based on the events.
\end{DoxyItemize}\hypertarget{README.md_autotoc_md9809}{}\doxysubsubsubsection{\texorpdfstring{Serializer Design}{Serializer Design}}\label{README.md_autotoc_md9809}
The serializer takes a data structure (the {\ttfamily DOMDocument}) and transforms it into a character representation -- an HTML5 document.

The serializer is broken into three parts\+:


\begin{DoxyItemize}
\item The {\ttfamily Output\+Rules} contain the rules to turn DOM elements into strings. The rules are an implementation of the interface {\ttfamily Rules\+Interface} allowing for different rule sets to be used.
\item The {\ttfamily Traverser}, which is a special-\/purpose tree walker. It visits each node node in the tree and uses the {\ttfamily Output\+Rules} to transform the node into a string.
\item {\ttfamily HTML5} manages the {\ttfamily Traverser} and stores the resultant data in the correct place.
\end{DoxyItemize}

The serializer ({\ttfamily save()}, {\ttfamily save\+HTML()}) follows the \href{http://www.w3.org/TR/2012/CR-html5-20121217/syntax.html\#serializing-html-fragments}{\texttt{section 8.\+9 of the HTML 5.\+0 spec}}. So tags are serialized according to these rules\+:


\begin{DoxyItemize}
\item A tag with children\+: \texorpdfstring{$<$}{<}foo\texorpdfstring{$>$}{>}CHILDREN\texorpdfstring{$<$}{<}/foo\texorpdfstring{$>$}{>}
\item A tag that cannot have content\+: \texorpdfstring{$<$}{<}foo\texorpdfstring{$>$}{>} (no closing tag)
\item A tag that could have content, but doesn\textquotesingle{}t\+: \texorpdfstring{$<$}{<}foo\texorpdfstring{$>$}{>}\texorpdfstring{$<$}{<}/foo\texorpdfstring{$>$}{>}
\end{DoxyItemize}\hypertarget{README.md_autotoc_md9810}{}\doxysubsubsection{\texorpdfstring{Known Issues (Or, Things We Designed Against the Spec)}{Known Issues (Or, Things We Designed Against the Spec)}}\label{README.md_autotoc_md9810}
Please check the issue queue for a full list, but the following are issues known issues that are not presently on the roadmap\+:


\begin{DoxyItemize}
\item Namespaces\+: HTML5 only \href{http://www.w3.org/TR/html5/infrastructure.html\#namespaces}{\texttt{supports a selected list of namespaces}} and they do not operate in the same way as XML namespaces. A {\ttfamily \+:} has no special meaning. By default the parser does not support XML style namespaces via {\ttfamily \+:}; to enable the XML namespaces see the XML Namespaces section
\item Scripts\+: This parser does not contain a Java\+Script or a CSS interpreter. While one may be supplied, not all features will be supported.
\item Reentrance\+: The current parser is not re-\/entrant. (Thus you can\textquotesingle{}t pause the parser to modify the HTML string mid-\/parse.)
\item Validation\+: The current tree builder is {\bfseries{not}} a validating parser. While it will correct some HTML, it does not check that the HTML conforms to the standard. (Should you wish, you can build a validating parser by extending DOMTree or building your own Event\+Handler implementation.)
\begin{DoxyItemize}
\item There is limited support for insertion modes.
\item Some autocorrection is done automatically.
\item Per the spec, many legacy tags are admitted and correctly handled, even though they are technically not part of HTML5.
\end{DoxyItemize}
\item \doxylink{class_attribute}{Attribute} names and values\+: Due to the implementation details of the PHP implementation of DOM, attribute names that do not follow the XML 1.\+0 standard are not inserted into the DOM. (Effectively, they are ignored.) If you\textquotesingle{}ve got a clever fix for this, jump in!
\item Processor Instructions\+: The HTML5 spec does not allow processor instructions. We do. Since this is a server-\/side library, we think this is useful. And that means, dear reader, that in some cases you can parse the HTML from a mixed PHP/\+HTML document. This, however, is an incidental feature, not a core feature.
\item HTML manifests\+: Unsupported.
\item PLAINTEXT\+: Unsupported.
\item Adoption Agency Algorithm\+: Not yet implemented. (8.\+2.\+5.\+4.\+7)
\end{DoxyItemize}\hypertarget{README.md_autotoc_md9811}{}\doxysubsubsection{\texorpdfstring{XML Namespaces}{XML Namespaces}}\label{README.md_autotoc_md9811}
