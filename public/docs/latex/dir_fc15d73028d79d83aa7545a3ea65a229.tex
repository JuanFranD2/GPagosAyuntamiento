\hypertarget{README.md_autotoc_md7705}{}\doxysubsection{\texorpdfstring{tldts -\/ Blazing Fast URL Parsing}{tldts -\/ Blazing Fast URL Parsing}}\label{README.md_autotoc_md7705}
{\ttfamily tldts} is a Java\+Script library to extract hostnames, domains, public suffixes, top-\/level domains and subdomains from URLs.

{\bfseries{Features}}\+:


\begin{DoxyEnumerate}
\item Tuned for {\bfseries{performance}} (order of 0.\+1 to 1 Î¼s per input)
\item Handles both URLs and hostnames
\item Full Unicode/\+IDNA support
\item Support parsing email addresses
\item Detect IPv4 and IPv6 addresses
\item Continuously updated version of the public suffix list
\item {\bfseries{Type\+Script}}, ships with {\ttfamily umd}, {\ttfamily esm}, {\ttfamily cjs} bundles and {\itshape type definitions}
\item Small bundles and small memory footprint
\item Battle tested\+: full test coverage and production use
\end{DoxyEnumerate}\hypertarget{README.md_autotoc_md7706}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{README.md_autotoc_md7706}

{\ttfamily tldts} methods {\ttfamily get\+Domain} and {\ttfamily get\+Subdomain} are designed to {\bfseries{work only with {\itshape known and valid} TLDs}}. This way, you can trust what a domain is.

{\ttfamily localhost} is a valid hostname but not a TLD. You can pass additional options to each method exposed by {\ttfamily tldts}\+:

\hypertarget{README.md_autotoc_md7717}{}\doxysubsubsection{\texorpdfstring{Updating the TLDs List}{Updating the TLDs List}}\label{README.md_autotoc_md7717}
{\ttfamily tldts} made the opinionated choice of shipping with a list of suffixes directly in its bundle. There is currently no mechanism to update the lists yourself, but we make sure that the version shipped is always up-\/to-\/date.

If you keep {\ttfamily tldts} updated, the lists should be up-\/to-\/date as well!\hypertarget{README.md_autotoc_md7718}{}\doxysubsection{\texorpdfstring{Performance}{Performance}}\label{README.md_autotoc_md7718}
{\ttfamily tldts} is the {\itshape fastest Java\+Script library} available for parsing hostnames. It is able to parse {\itshape millions of inputs per second} (typically 2-\/3M depending on your hardware and inputs). It also offers granular options to fine-\/tune the behavior and performance of the library depending on the kind of inputs you are dealing with (e.\+g.\+: if you know you only manipulate valid hostnames you can disable the hostname extraction step with {\ttfamily \{ extract\+Hostname\+: false \}}).
