\hypertarget{README.md_autotoc_md1078}{}\doxysubsubsection{\texorpdfstring{Using only global styles or only classes}{Using only global styles or only classes}}\label{README.md_autotoc_md1078}
Although we recommend thinking of this plugin as a "{}form reset"{} rather than a collection of form component styles, in some cases our default approach may be too heavy-\/handed, especially when integrating this plugin into existing projects.

If generating both the global (base) styles and classes doesn\textquotesingle{}t work well with your project, you can use the {\ttfamily strategy} option to limit the plugin to just one of these approaches.


\begin{DoxyCode}{0}
\DoxyCodeLine{//\ tailwind.config.js}
\DoxyCodeLine{plugins:\ [}
\DoxyCodeLine{\ \ require("{}@tailwindcss/forms"{})(\{}
\DoxyCodeLine{\ \ \ \ strategy:\ 'base',\ //\ only\ generate\ global\ styles}
\DoxyCodeLine{\ \ \ \ strategy:\ 'class',\ //\ only\ generate\ classes}
\DoxyCodeLine{\ \ \}),}
\DoxyCodeLine{],}

\end{DoxyCode}


When using the {\ttfamily base} strategy, form elements are styled globally, and no {\ttfamily form-\/\{name\}} classes are generated.

When using the {\ttfamily class} strategy, form elements are not styled globally, and instead must be styled using the generated {\ttfamily form-\/\{name\}} classes.
