\doxysubsection{Detailed Description}
Autoloader checking if the class is really defined in the file found.

The Class\+Loader will wrap all registered autoloaders and will throw an exception if a file is found but does not declare the class.

It can also patch classes to turn docblocks into actual return types. This behavior is controlled by the SYMFONY\+\_\+\+PATCH\+\_\+\+TYPE\+\_\+\+DECLARATIONS env var, which is a url-\/encoded array with the follow parameters\+:
\begin{DoxyItemize}
\item "{}force"{}\+: any value enables deprecation notices -\/ can be any of\+:
\begin{DoxyItemize}
\item "{}phpdoc"{} to patch only docblock annotations
\item "{}2"{} to add all possible return types
\item "{}1"{} to add return types but only to tests/final/internal/private methods
\end{DoxyItemize}
\item "{}php"{}\+: the target version of PHP -\/ e.\+g. "{}7.\+1"{} doesn\textquotesingle{}t generate "{}object"{} types
\item "{}deprecations"{}\+: "{}1"{} to trigger a deprecation notice when a child class misses a return type while the parent declares an "{}@return"{} annotation
\end{DoxyItemize}

Note that patching doesn\textquotesingle{}t care about any coding style so you\textquotesingle{}d better to run php-\/cs-\/fixer after, with rules "{}phpdoc\+\_\+trim\+\_\+consecutive\+\_\+blank\+\_\+line\+\_\+separation"{} and "{}no\+\_\+superfluous\+\_\+phpdoc\+\_\+tags"{} enabled typically.
