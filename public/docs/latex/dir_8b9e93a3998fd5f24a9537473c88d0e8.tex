\hypertarget{README.md_autotoc_md7748}{}\doxysubsubsection{\texorpdfstring{Cookie Prefixes}{Cookie Prefixes}}\label{README.md_autotoc_md7748}
\doxylink{class_cookie}{Cookie} prefixes are a way to indicate that a given cookie was set with a set of attributes simply by inspecting the first few characters of the cookie\textquotesingle{}s name.

Two prefixes are defined\+:


\begin{DoxyItemize}
\item {\ttfamily "{}\+\_\+\+\_\+\+Secure-\/"{}}

If a cookie\textquotesingle{}s name begins with a case-\/sensitive match for the string {\ttfamily \+\_\+\+\_\+\+Secure-\/}, then the cookie was set with a "{}\+Secure"{} attribute.
\item {\ttfamily "{}\+\_\+\+\_\+\+Host-\/"{}}

If a cookie\textquotesingle{}s name begins with a case-\/sensitive match for the string {\ttfamily \+\_\+\+\_\+\+Host-\/}, then the cookie was set with a "{}\+Secure"{} attribute, a "{}\+Path"{} attribute with a value of "{}/"{}, and no "{}\+Domain"{} attribute.
\end{DoxyItemize}

If {\ttfamily prefix\+Security} is enabled for {\ttfamily Cookie\+Jar}, then cookies that match the prefixes defined above but do not obey the attribute restrictions are not added.

You can define this functionality by passing in the {\ttfamily prefix\+Security} option to {\ttfamily Cookie\+Jar}. It can be one of 3 values\+:


\begin{DoxyEnumerate}
\item {\ttfamily silent}\+: ({\bfseries{default}}) Enable cookie prefix checking but silently fail to add the cookie if conditions are not met.
\item {\ttfamily strict}\+: Enable cookie prefix checking and error out if conditions are not met.
\item {\ttfamily unsafe-\/disabled}\+: Disable cookie prefix checking.
\end{DoxyEnumerate}

\begin{quote}
If {\ttfamily ignore\+Error} is passed in as {\ttfamily true} when setting a cookie then the error is silent regardless of the {\ttfamily prefix\+Security} option (assuming it\textquotesingle{}s enabled). \end{quote}
